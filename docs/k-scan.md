# K-SCAN 가이드

## 한눈에 보기

K-SCAN은 KOMPASS에서 얻은 검색 결과를 캡처하고, 청구항과 각 문헌이 얼마나 가까운지 점수와 설명으로 보여주는 모듈입니다. 쉽게 말하면 "검색은 했는데, 나온 문헌 중 무엇을 먼저 볼지"를 빠르게 정리해 주는 역할입니다.

K-Query가 검색식을 만드는 모듈이라면, K-SCAN은 그 검색식으로 얻은 결과를 정리하고 우선순위를 매기는 모듈입니다.

## 언제 쓰면 좋은가

- KOMPASS 검색 결과가 많이 나와서 무엇부터 봐야 할지 판단이 어려울 때
- 검색 결과 중 점수가 높은 문헌만 우선 검토하고 싶을 때
- 출원발명과 비교했을 때 어떤 문헌이 더 가까운지 빠르게 보고 싶을 때
- 다음 검색식 수정에 참고할 평가 결과가 필요할 때

## 이 모듈이 하는 일

- KOMPASS 화면에서 필요한 응답을 자동 캡처합니다.
- 출원발명과 인용문헌의 내용을 비교해 유사도 점수와 설명을 만듭니다.
- 평가 결과를 누적 저장하고, 다시 볼 수 있게 결과 창으로 정리합니다.
- 선택한 문헌의 출원번호를 추출하거나, 특정 항목만 다시 평가할 수 있게 지원합니다.
- 평가 결과를 K-Query나 K-Research의 후속 보정에 활용할 수 있게 연결합니다.

## 준비물

- 출원발명 또는 청구항 텍스트
- KOMPASS가 열린 웹 탭
- 공통 설정에 저장된 OpenWebUI 주소와 API 키

있으면 더 좋은 것:

- K-Query에서 정리한 청구항
- 검색 결과를 어떤 기준으로 줄이거나 넓힐지에 대한 초안

## 사용 흐름

1. KOMPASS 탭에서 K-SCAN을 엽니다.
2. 출원발명 또는 청구항 텍스트를 확인합니다.
3. 필요하면 K-Query에서 공유된 청구항을 가져옵니다.
4. 캡처 시작을 누릅니다.
5. KOMPASS에서 검색 결과를 확인하거나 이동합니다.
6. K-SCAN이 문헌과 요약 정보를 수집하고 평가합니다.
7. 별도 결과 창에서 점수, 설명, 세부 내용을 확인합니다.
8. 필요한 문헌만 골라 다시 평가하거나 출원번호를 추출합니다.

## 화면 구성

### 1. 사이드패널

사이드패널은 실시간 실행용 화면입니다.

- 출원발명 입력
- K-Query 청구항 가져오기
- 평가에 사용할 프롬프트 템플릿 확인
- 캡처 시작 / 중지
- 현재 상태 확인

즉, "무엇을 기준으로 어떤 검색 결과를 평가할지"를 준비하고 실행하는 곳입니다.

### 2. 결과 창

결과 창은 누적된 평가를 보는 화면입니다.

- 최신 평가 점수
- 전체 이력 목록
- 점수 기준 필터링
- 정렬과 검색
- 행 선택 후 출원번호 추출
- 선택 항목 재평가
- 개별 항목 상세 보기

즉, "무엇이 높은 우선순위 문헌인지"를 정리해서 보는 곳입니다.

## 결과물

K-SCAN이 제공하는 핵심 결과는 다음과 같습니다.

- 문헌별 점수
- 점수에 대한 설명
- 최신 결과와 누적 이력
- 선택 문헌 목록과 출원번호 추출 결과
- 후속 검색식 보정에 참고할 평가 근거

## 다른 모듈과의 연결

- `K-Query`
  - 청구항을 가져와 같은 기준으로 결과를 평가할 수 있습니다.
  - K-SCAN의 평가 결과는 검색식 수정 방향을 잡는 데 도움을 줍니다.
- `K-Research`
  - 자동 탐색 루프에서도 K-SCAN 계열의 캡처와 평가 흐름이 활용됩니다.
- `K-LARC`
  - 점수가 높거나 중요한 문헌을 추려 K-LARC의 정밀 인용 분석으로 넘길 수 있습니다.

## 잘 맞는 사용 방식

- 처음에는 점수 상위 문헌부터 빠르게 훑고, 그다음 중간 점수 문헌을 선별 검토하는 방식이 효율적입니다.
- 점수만 보지 말고, 왜 그 점수가 나왔는지 설명까지 같이 확인해야 합니다.
- 필요 없는 문헌을 무작정 많이 보는 대신, 결과 창에서 선별 후 다음 단계로 넘기는 것이 좋습니다.

## 주의할 점

- K-SCAN의 점수는 검토 우선순위를 잡아 주는 보조 지표이지, 최종 법적 판단 자체는 아닙니다.
- KOMPASS 탭 문맥이 맞지 않으면 기대한 캡처가 이뤄지지 않을 수 있습니다.
- 입력 청구항이 불명확하면 점수 해석도 흔들릴 수 있으므로, 가능하면 K-Query나 정제된 청구항을 함께 사용하는 편이 좋습니다.
- 자동 캡처 결과는 누적되므로, 어떤 세션의 결과인지 구분하면서 보는 습관이 필요합니다.

## 이런 사용자에게 특히 유용하다

- 검색 결과가 많아서 1차 우선순위 분류가 필요한 사용자
- 검색식 수정 전에 실제 결과 품질을 빠르게 판단하고 싶은 사용자
- 유사도 평가와 출원번호 추출을 한 화면 흐름으로 처리하고 싶은 사용자
