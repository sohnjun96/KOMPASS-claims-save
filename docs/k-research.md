# K-Research 가이드

## 한눈에 보기

K-Research는 검색식 생성, KOMPASS 수집, 문헌 평가, 검색식 보정을 자동으로 반복하는 탐색 모듈입니다. 사람이 매번 검색식을 손으로 고치고 다시 검색하는 과정을 줄여 주는, K-SUITE의 자동 탐색 관제판이라고 보면 됩니다.

K-Query가 검색식의 시작점을 만든다면, K-Research는 그 검색식을 여러 번 다듬어 가며 더 나은 후보 문헌을 찾는 역할을 맡습니다.

## 언제 쓰면 좋은가

- 한 번의 검색으로 만족스러운 문헌이 잘 나오지 않을 때
- 결과가 너무 많거나 너무 적어서 반복 조정이 필요할 때
- 검색식 보정을 일정 부분 자동화하고 싶을 때
- 단일 후보뿐 아니라 조합 가능한 문헌 후보까지 보고 싶을 때

## 이 모듈이 하는 일

- 청구항에서 초기 검색식을 만듭니다.
- KOMPASS에서 결과를 수집합니다.
- 수집된 문헌을 평가하고 점수와 커버리지를 봅니다.
- 결과가 너무 많음, 너무 적음, 적정 수준인지 판단합니다.
- 그 판단에 따라 검색식을 넓히거나 좁히거나 균형 조정합니다.
- 종료 조건을 만족할 때까지 이 과정을 반복합니다.

## 준비물

- 청구항 텍스트
- KOMPASS가 열린 웹 탭
- 공통 설정에 저장된 OpenWebUI 주소와 API 키

있으면 좋은 것:

- K-Query에서 만든 초기 검색식
- K-SCAN에서 얻은 기존 평가 감각

## 기본 사용 흐름

1. `실행` 탭에서 청구항을 입력합니다.
2. 필요하면 K-Query 또는 K-SCAN에서 청구항을 가져옵니다.
3. 초기 검색식을 생성합니다.
4. 자동 탐색 시작을 누릅니다.
5. 상단 상태바와 상태 카드에서 현재 단계와 다음 행동을 확인합니다.
6. 자동 루프가 수집과 평가를 반복합니다.
7. 필요 시 사람이 결과 많음, 적음, 적정 중 하나를 판단해 개입합니다.
8. `결과` 탭에서 반복 결과와 후보 문헌을 확인합니다.

## 화면 구성

### 1. 실행

기본 작업면입니다.

- 자동 상태
- 현재 단계
- 현재 검색식
- 반복 횟수
- 최근 결과 수
- 다음 행동
- 청구항 입력과 초기 검색식 생성
- 사람 확인 필요 카드

즉, 자동 탐색이 지금 무엇을 하고 있는지 실시간으로 보는 곳입니다.

### 2. 결과

이번 반복의 결론을 모아 보는 작업면입니다.

- 평가 건수
- 점수 분포
- 최고 점수와 커버리지
- 종료 판정
- 상위 문헌 후보
- 조합 후보
- 검색식이 왜 바뀌었는지에 대한 근거

즉, 자동 루프가 낸 결론을 빠르게 읽는 곳입니다.

### 3. 고급

예외 대응과 운영자 제어를 위한 작업면입니다.

- 수동 검색 또는 루프 제어
- KOMPASS 제어 보조
- 결과 수 판단 버튼
- 검색식 원문 편집
- 롤백
- 진단 정보
- 피드백 이력

기본 사용자는 자주 볼 필요가 없지만, 자동 루프가 애매한 상황에서 유용합니다.

## 자동 탐색이 특히 유용한 이유

K-Research의 강점은 "한 번의 검색"이 아니라 "반복 개선"에 있습니다.

- 결과가 너무 많으면 검색식을 더 좁힙니다.
- 결과가 너무 적으면 검색식을 더 넓힙니다.
- 어느 정도 적정하면 좋은 후보를 중심으로 평가를 이어갑니다.
- 중복되거나 의미 없는 방향으로 검색식이 흐르지 않도록 보정합니다.

즉, 사람이 반복해서 하던 탐색 조정 업무를 체계화한 모듈입니다.

## 결과물

K-Research는 단순한 검색식 하나만 남기지 않습니다. 보통 다음 정보가 함께 쌓입니다.

- 반복마다 달라진 검색식
- 문헌 평가 결과
- 점수와 커버리지 요약
- 종료 여부와 종료 이유
- 다음 반복에서 왜 검색식을 바꿨는지에 대한 근거

이 결과는 이후 K-LARC에서 최종 후보를 정밀 분석하는 데 연결하기 좋습니다.

## 다른 모듈과의 연결

- `K-Query`
  - K-Research의 출발점이 되는 초기 검색식과 청구항 구조를 제공합니다.
- `K-SCAN`
  - 문헌 수집과 평가의 감각을 공유하는 가까운 모듈입니다.
- `K-LARC`
  - K-Research에서 찾은 후보 문헌을 최종 정밀 분석 단계로 넘기는 연결점입니다.

## 잘 맞는 사용 방식

- 자동 탐색을 시작하기 전, 청구항 자체는 최대한 명확하게 정리해 두는 것이 좋습니다.
- 사람이 개입할 때는 "결과 수 판단"만 명확히 해 줘도 전체 흐름이 많이 안정됩니다.
- 반복마다 검색식을 전부 새로 읽기보다, 직전 대비 무엇이 추가되고 제거됐는지를 중심으로 보는 편이 효율적입니다.

## 주의할 점

- 자동 탐색이 있다고 해서 무조건 정답 문헌을 바로 찾는 것은 아닙니다. 이 모듈은 탐색 효율을 높이는 도구입니다.
- KOMPASS 탭 문맥이 맞지 않으면 수집 단계가 기대대로 동작하지 않을 수 있습니다.
- 자동 루프가 멈춘 경우는 실패만이 아니라 사람이 판단해야 할 분기점일 수도 있습니다.
- 결과 탭의 수치와 후보는 매우 유용하지만, 최종 법적 판단은 여전히 사람 검토가 필요합니다.

## 이런 사용자에게 특히 유용하다

- 검색식 수정과 재검색을 반복하는 시간이 큰 사용자
- 한 번의 수동 검색보다 반복 탐색 전략이 필요한 사용자
- 검색식 보정 이유와 후보 문헌 변화를 함께 관리하고 싶은 사용자
