# K-Query 가이드

## 한눈에 보기

K-Query는 청구항 문장을 검색에 적합한 검색식으로 바꾸는 모듈입니다. 사람이 직접 긴 검색식을 처음부터 만드는 대신, 청구항의 핵심 요소를 구조화해서 검색식 초안을 만들고, 필요하면 범위를 넓히거나 좁히면서 다듬는 역할을 합니다.

K-SUITE 안에서는 보통 가장 먼저 사용하는 모듈입니다. 이후 단계에서 K-SCAN이나 K-Research가 검색 결과를 평가하면, 그 피드백을 다시 받아 검색식을 더 정교하게 조정하는 출발점이 됩니다.

## 언제 쓰면 좋은가

- 청구항을 바탕으로 첫 검색식을 빠르게 만들고 싶을 때
- 검색식이 너무 짧거나 너무 넓어서 검색 품질이 낮을 때
- 어떤 용어를 중심으로 검색해야 할지 감이 잘 오지 않을 때
- 검색식 초안과 수정 이력을 함께 관리하고 싶을 때

## 이 모듈이 하는 일

- 청구항을 검색 가능한 핵심 요소들로 나눕니다.
- 각 요소에 대해 비슷한 표현, 대체 표현, 관련 표현을 확장합니다.
- 너무 일반적이거나 잡음을 늘릴 표현은 걸러냅니다.
- 최종적으로 사람이 바로 검토할 수 있는 검색식 형태로 조합합니다.
- 생성 결과와 수정 이력을 저장해 이후 비교와 재사용이 가능하게 합니다.

## 준비물

- 분석하려는 청구항 텍스트
- 공통 설정에 저장된 OpenWebUI 주소와 API 키

추가로 있으면 좋은 것:

- 이전 검색 결과에서 얻은 피드백
- 특정 기술 분야에서 반드시 포함하거나 제외해야 할 용어 기준

## 사용 흐름

1. 청구항을 입력합니다.
2. 검색 범위를 어느 정도로 가져갈지 조정합니다.
3. 검색식 생성을 실행합니다.
4. 생성된 검색식을 검토합니다.
5. 필요하면 용어를 다듬거나 다시 생성합니다.
6. 최종 검색식을 복사해 KOMPASS나 다음 모듈에서 사용합니다.

## 화면에서 보게 되는 것

### 1. 청구항 입력 영역

- 청구항 원문을 넣는 곳입니다.
- 예시 문장을 불러오거나 입력을 비울 수 있습니다.
- 입력 길이와 현재 상태를 함께 확인할 수 있습니다.

### 2. 검색식 결과 영역

- 생성된 최종 검색식을 보여줍니다.
- 바로 복사하거나 내보낼 수 있습니다.
- 검색 범위 조절 결과가 함께 반영됩니다.

### 3. 실행 상태 영역

- 생성 과정이 어느 단계까지 진행됐는지 보여줍니다.
- 분석, 표현 확장, 최종 조합 같은 큰 흐름을 사용자 기준으로 이해할 수 있게 구성되어 있습니다.

### 4. 용어 검토 영역

- 핵심 요소별 표현을 검토하고 조정하는 구간입니다.
- 어떤 표현이 유지되고 어떤 표현이 빠졌는지 확인할 수 있습니다.

### 5. 로그 및 고급 확인 영역

- 생성 과정에서 어떤 판단이 이뤄졌는지 추적할 수 있습니다.
- 일반 사용자는 꼭 볼 필요는 없지만, 검색식 품질을 세밀하게 조정할 때 유용합니다.

## 결과물

K-Query의 가장 중요한 결과물은 다음 두 가지입니다.

- 실제 검색에 넣을 최종 검색식
- 생성 과정과 수정 이력을 담은 검색식 작업 기록

이 기록은 이후 K-SCAN의 평가 결과나 K-Research의 자동 탐색 결과와 연결될 수 있습니다.

## 다른 모듈과의 연결

- `K-SCAN`
  - K-Query에서 만든 청구항과 검색식을 바탕으로 실제 검색 결과를 평가합니다.
- `K-Research`
  - K-Query가 시작점을 만든다면, K-Research는 그 검색식을 반복적으로 보정합니다.
- `K-LARC`
  - K-Query에서 다룬 청구항 구조는 이후 인용 분석 단계의 기초 자료가 됩니다.

## 잘 맞는 사용 방식

- 먼저 너무 완벽한 검색식을 만들려고 하지 말고, 핵심 요소가 빠지지 않는 초안을 만듭니다.
- 첫 결과가 넓으면 범위를 줄이고, 너무 적으면 표현을 넓히는 식으로 단계적으로 조정합니다.
- 검색식만 복사해서 끝내지 말고, 어떤 논리로 만들어졌는지 함께 확인해 두는 것이 좋습니다.

## 주의할 점

- 검색식이 기술적으로 그럴듯해 보여도 실제 검색 결과 품질은 별도로 검증해야 합니다.
- 표현을 과도하게 늘리면 결과 수는 많아지지만 잡음도 커질 수 있습니다.
- 반대로 표현을 너무 엄격하게 묶으면 중요한 문헌을 놓칠 수 있습니다.
- K-Query는 검색식을 잘 만드는 도구이지, 검색 결과의 적합성을 최종 판단하는 도구는 아닙니다. 그 판단은 K-SCAN이나 K-LARC 단계에서 이어집니다.

## 이런 사용자에게 특히 유용하다

- 청구항을 검색식으로 바꾸는 시간을 줄이고 싶은 심사 지원 실무자
- 검색식을 감에 의존하지 않고 구조적으로 만들고 싶은 사용자
- 한 번 만든 검색식을 기록으로 남기고, 다음 반복에서 개선하고 싶은 사용자
