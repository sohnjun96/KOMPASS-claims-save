# K-LARC 가이드

## 한눈에 보기

K-LARC는 청구항과 인용문헌을 정밀하게 비교해, 어떤 구성요소가 어느 문헌에 대응되는지 구조적으로 분석하는 모듈입니다. K-SUITE 안에서 가장 본격적인 분석 작업을 맡는 화면이라고 보면 됩니다.

단순히 "비슷하다"는 수준을 넘어서, 청구항의 각 요소가 어떤 문헌의 어느 부분에서 확인되는지, 부족한 부분은 무엇인지, 의견제출통지서용 정리까지 어떻게 이어질 수 있는지를 한 작업 공간 안에서 다룹니다.

## 언제 쓰면 좋은가

- 선별된 인용문헌을 청구항 기준으로 깊게 비교해야 할 때
- 어떤 문헌이 어느 요소를 충족하는지 표 형태로 보고 싶을 때
- 단일 문헌이 아닌 복수 문헌 조합 가능성까지 검토해야 할 때
- 의견제출통지서 초안이나 검토용 정리표가 필요할 때

## 이 모듈이 하는 일

- 청구항을 구성요소 단위로 분해합니다.
- 각 인용문헌에서 대응되는 근거를 찾고 정리합니다.
- 요소별 충족 여부와 주의 사항을 검토합니다.
- 필요한 경우 부족한 근거를 다시 보완 탐색합니다.
- 최종적으로 분석 결과, 근거 검토, 의견제출통지서용 정리, 문헌 Q&A까지 하나의 작업 흐름으로 묶습니다.

## 준비물

- 분석할 청구항
- 검토할 인용문헌들
- 공통 설정에 저장된 API 키

권장 준비:

- K-SCAN에서 1차 선별된 문헌
- 문헌 이름을 명확히 구분할 수 있는 목록

## 사용 흐름

1. 청구항을 입력하거나 가져옵니다.
2. 분석할 인용문헌을 추가합니다.
3. 빠른 분석 또는 정밀 분석 중 적합한 방식을 선택합니다.
4. 분석을 실행합니다.
5. 결과 표에서 요소별 대응 관계를 확인합니다.
6. 의견제출통지서용 정리 화면에서 필요한 형식으로 재구성합니다.
7. 필요하면 문헌 Q&A나 디버그 화면으로 추가 검토합니다.

## 화면 구성

### 1. 청구항 영역

- 분석 대상 청구항을 관리하는 곳입니다.
- 여러 청구항을 다룰 수 있고, K-SCAN에서 공유된 청구항을 불러올 수도 있습니다.

### 2. 인용문헌 영역

- 분석할 문헌을 추가하는 곳입니다.
- 탭에서 가져오기, PDF 업로드, 직접 입력 같은 방식이 가능합니다.
- 여러 문헌을 나란히 비교할 수 있게 준비하는 단계입니다.

### 3. 분석 실행 영역

- 빠른 분석과 정밀 분석을 선택할 수 있습니다.
- 빠른 분석은 큰 방향을 빨리 보는 데 적합합니다.
- 정밀 분석은 근거와 검증까지 포함한 본격 작업에 적합합니다.

### 4. 분석 결과 작업면

분석 결과 작업면은 실제 비교 분석의 중심입니다.

- 청구항 요소별 대응 관계
- 문헌별 근거 내용
- 위치 정보와 일치 유형
- 검증 또는 주의 표시

즉, 청구항과 문헌을 가장 직접적으로 대조하는 영역입니다.

### 5. 의견제출통지서 작업면

- 분석 결과를 문서 작성 관점으로 다시 정리하는 곳입니다.
- 어떤 문헌을 주된 근거로 쓸지, 어떤 요소가 어떻게 대응되는지 표 형태로 볼 수 있습니다.
- 복사 가능한 형식으로 정리하기 좋게 구성되어 있습니다.

### 6. 문헌 Q&A 작업면

- 특정 인용문헌에 대해 질문하고 답을 받는 보조 공간입니다.
- 차이점 설명, 특정 요소 확인, 문헌 간 비교 같은 보조 검토에 유용합니다.

### 7. 디버그 작업면

- 내부 분석 과정을 단계별로 확인하는 공간입니다.
- 일반 사용자보다 검토 품질을 세밀하게 확인하려는 사용자에게 유용합니다.

## 결과물

K-LARC에서 얻게 되는 대표 결과는 다음과 같습니다.

- 청구항 요소별 분석 표
- 문헌별 대응 근거
- 요소별 충족 여부 정리
- 검토용 주의/경고 정보
- 의견제출통지서용 표 형식 결과
- 문헌 기반 질의응답 결과

## 다른 모듈과의 연결

- `K-SCAN`
  - K-SCAN에서 선별한 문헌을 K-LARC에서 정밀 분석 대상으로 삼기 좋습니다.
- `K-Query`
  - K-Query에서 다뤘던 청구항 구조가 K-LARC의 분석 출발점이 됩니다.
- `K-Research`
  - K-Research로 탐색 범위를 넓힌 뒤, 최종 후보를 K-LARC에서 깊게 비교하는 흐름이 자연스럽습니다.

## 잘 맞는 사용 방식

- 처음부터 너무 많은 문헌을 넣기보다, 1차 후보를 줄인 뒤 분석하는 편이 좋습니다.
- 빠른 분석으로 큰 방향을 먼저 보고, 필요한 건만 정밀 분석으로 넘기면 효율적입니다.
- 의견제출통지서 작업면은 결과를 재정리하는 공간이므로, 먼저 분석 결과 표를 충분히 검토한 뒤 사용하는 것이 좋습니다.

## 주의할 점

- K-LARC는 가장 정밀한 분석 모듈이지만, 입력 문헌 품질이 낮거나 문헌이 너무 많으면 검토 부담도 함께 커집니다.
- 단일 문헌 분석과 복수 문헌 조합 검토는 목적이 다르므로, 어떤 결론을 얻고 싶은지 먼저 정리하는 것이 좋습니다.
- 결과 표에서 보이는 대응 관계는 문서 작성과 검토를 돕는 구조화 결과이므로, 최종 심사 판단은 반드시 사람이 다시 확인해야 합니다.

## 이런 사용자에게 특히 유용하다

- 청구항 대 문헌 비교를 체계적으로 정리하고 싶은 사용자
- 의견제출통지서용 표와 근거를 한 흐름으로 정리하고 싶은 사용자
- 단순 검색을 넘어서 근거 수준의 정밀 비교가 필요한 사용자
